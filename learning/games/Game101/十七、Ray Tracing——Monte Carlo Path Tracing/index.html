# 蒙特卡洛积分（Monte Carlo Integration）

并不对整个区间求积分，而是在整个区间内做多次均匀的采样。

求得采样的平均值，并把平均值视为区域任何点的 f(x)f(x)f(x) 值（不规则曲线看成直线）。

# Whitted-Style Ray Tracing is Wrong

Whitted-Style 的光线追踪在效果上来说，是不理想的，但是其本身的渲染方程是正确的。

但这个方程的求解本身就比较的困难：

  • 计算过程是递归的。
  • 每次计算要对一个半球求积分。

Lo(p,wo)=Le(p,wo)+∫Ω+Li(p,wi)fr(p,wi,wo)(n⋅wi)dwiL_o(p,w_o) = L_e(p,w_o) + \int_{\Omega^+}L_i(p, w_i)f_r(p,w_i,w_o)(n \cdot w_i) dw_iLo​(p,wo​)=Le​(p,wo​)+∫Ω+​Li​(p,wi​)fr​(p,wi​,wo​)(n⋅wi​)dwi​

# A Simple Monte Carlo Solution

假设只渲染场景中的一个着色点。

假设该点本身是不发光的，所以计算公公式内，不需要计算 Le(p,wo)L_e(p,w_o)Le​(p,wo​)

Lo(p,wo)=∫Ω+Li(p,wi)fr(p,wi,wo)(n⋅wi)dwiL_o(p,w_o) = \int_{\Omega^+}L_i(p, w_i)f_r(p,w_i,w_o)(n \cdot w_i) dw_iLo​(p,wo​)=∫Ω+​Li​(p,wi​)fr​(p,wi​,wo​)(n⋅wi​)dwi​

所以，计算物体本身在某个方向的光线，需要对物体在整个半球所接受到的光线求积分。

这时候，「蒙特卡洛积分」就闪亮登场了🌟，积分对于整体的计算还是过于耗了，因此进行一个简化：

  • 采样某几个点的结果的均值作为最终的积分结果不失为一个不错的选择。

Monte Carlo Intergration:∫abf(x)dx≈1N∑k=1Nf(Xk)p(Xk)Xk∼p(x)\text{Monte Carlo Intergration:} \int_a^b f(x)dx \approx \frac{1}{N} \sum_{k=1}^{N} \frac{f(X_k)}{p(X_k)} \quad X_k \thicksim p(x)Monte Carlo Intergration:∫ab​f(x)dx≈N1​k=1∑N​p(Xk​)f(Xk​)​Xk​∼p(x)

其中, f(x) =Li(p,wi)fr(p,wi,wo)(n⋅wi)\text{其中, f(x) = } L_i(p, w_i)f_r(p,w_i,w_o)(n \cdot w_i)其中, f(x) = Li​(p,wi​)fr​(p,wi​,wo​)(n⋅wi​)

PDF:pdf(ωi)=1/2π\text{PDF:} \quad pdf(\omega_i) = 1/ 2\piPDF:pdf(ωi​)=1/2π

因此，对于半球的积分，通过「蒙特卡洛积分」进行多次采样的计算公式可以写成如下形式：

Lo(p,wo)≈1N∑i=1NLi(p,wi)fr(p,wi,wo)(n⋅wi)pdf(wi)L_o(p,w_o) \approx \frac{1}{N} \sum_{i=1}^{N} \frac{L_i(p, w_i)f_r(p,w_i,w_o)(n \cdot w_i)}{pdf(w_i)}Lo​(p,wo​)≈N1​i=1∑N​pdf(wi​)Li​(p,wi​)fr​(p,wi​,wo​)(n⋅wi​)​

写成代码的形式：

  • 从着色点向半球的 N 个方向，射出 N 条光线
  • 如果光线最终和点光源有交点，那就计算该光线亮度Lo=Li∗fr∗cosθ/pdf(wi)L_o = {L_i * f_r * cos \theta} / { pdf(w_i)}Lo​=Li​∗fr​∗cosθ/pdf(wi​) 。
  • 对射出的 N 条光线亮度求平均

# Inreoducing Global Illumination

之前叙述的计算蒙特卡洛积分来求着色点问题，没有考虑光线射出后，遇到反射物体的情况。但是在处理全局光照的情况下，这是非常有必要的。

由于物体不仅仅可以接受来自光源的光线，还可以接受来自其他物体反射的光线，因此如果想要得到真实的全局光照，就应该对间接光照进行计算。例如点 Q 提供给点 P 的光照可以简单的视为吧点 Q 作为一个光源，从 P 点对 Q 点进行观察。

写成代码的形式，假设计算该点的着色函数为 shade(p,wo)shade(p, wo)shade(p,wo)：

  • 从着色点向半球的 N 个方向，射出 N 条光线。
  • 计算光线和光源交点。
    • 如果光线最终和点光源有交点，那就计算该光线亮度 Lo=Li∗fr∗cosθ/pdf(wi)L_o = {L_i * f_r * cos \theta} / { pdf(w_i)}Lo​=Li​∗fr​∗cosθ/pdf(wi​) 。
    • 如果和反射物体有交点，那就计算反射后的光线亮度 Lo=shade(q,wo)∗fr∗cosθ/pdf(wi)L_o = {shade(q, wo) * f_r * cos \theta} / { pdf(w_i)}Lo​=shade(q,wo)∗fr​∗cosθ/pdf(wi​) 。
  • 对射出的 N 条光线亮度求平均。

如果递归的计算，当射出的光线数量过多是，计算量级太过庞大。

有没有什么比较好的办法呢？—— 如果只采样 1 条光线。

但是只采样 1 条光线，误差会不会很大呢？因此，为了避免误差产生的噪声，一般会选择多个方向的光线做采样为 1 的「路径追踪」。

当 N 取 1 的情况下，为了降低误差，需要用多个光线穿过着色点进行采样。

  • 像素内随机取 N 个点
  • 从观察点射出 N 条光线，穿过视点，计算光线追踪。
    • 如果光线打到了物体，则计算该点的亮度。
  • 最终把单个像素内的所有点的亮度求平均

# 如果光线一直反射，怎么办

根据之前的计算规则，会计算每次光线反射后的。

但试想一下，如果一条光线一直在进行反射，那么是不是最终就无法计算源像素点的亮度了。

# 解决办法：Russian Roulette（RR）

给追踪一个概率，并非每次光线反射都进行计算。并且，为了避免光线的能量由于概率产生削减，对概率计算进行调整：

E=P∗(Lo/P)+(1−P)∗0=LoE = P * (L_o / P) + (1 -P) * 0 = L_oE=P∗(Lo​/P)+(1−P)∗0=Lo​

最终的计算公式 shade(p,wo)shade(p,wo)shade(p,wo) ：

定义光线发生折射或者反射的概率 P_RR∼[0,1]P\_RR \thicksim [0,1]P_RR∼[0,1]

  • 判断光线是否发生折射或者反射
    • 不发生，直接返回 0
    • 发生，如果光线击中物体
      • 物体如果是光源 Li∗fr∗cos⁡θ/pdf(wi)/P_RRL_i * f_r * \cos \theta / pdf(wi) / P\_RRLi​∗fr​∗cosθ/pdf(wi)/P_RR 。
      • 物体非光源 shade(q,−wi)∗fr∗cos⁡θ/pdf(wi)/P_RRshade(q, -wi) * f_r * \cos \theta / pdf(wi) / P\_RRshade(q,−wi)∗fr​∗cosθ/pdf(wi)/P_RR 。

# 思考

如果光源本身非常非常的小，就算从一个点打出 N 个光线，依然没有一个光线能够和光源有交集，这种情况下的计算将变得非常低效。

# 如何提高效率

为了提高采样效率，可以选择直接在光源上进行采样。

假设光源上是一个二维的平面，面积为 A。那么对 A 区域采样的 PDF 值为 1/A1/A1/A 。

那么在着色点计算半球积分，是否能转化为对光源区域 A 范围内的积分。

因此需要在 dwdwdw 和 dAdAdA 之间做个转换。又因为 dwdwdw 表示投影到单位圆上的面积，立体角定义，面积 / 距离平方。参考十六课。所以 dAdAdA 和 dwdwdw 关系如下

dw=dAcos⁡θ′∣∣x′−x∣∣2dw = \frac{dA \cos \theta^{\prime}}{|| x^{\prime} - x ||^2}dw=∣∣x′−x∣∣2dAcosθ′​

重新定义渲染方程，用以光源做积分的形式表示

最终的渲染方程变得异常的简单，只需要对光滑的表面进行积分，再对距离做一定的运算。

Lo(x,wo)=∫Ω+Li(x,wi)fr(x,wi,wo)cos⁡θdwi=∫ALi(x,wi)fr(x,wi,wo)cos⁡θcos⁡θ′∣∣x′−x∣∣2dAL_o(x,w_o) = \int_{\Omega^+}L_i(x, w_i)f_r(x,w_i,w_o) \cos \theta dw_i = \int_{A}L_i(x, w_i)f_r(x,w_i,w_o) \frac{\cos\theta \cos\theta^{\prime}}{|| x^{\prime} - x ||^2} dALo​(x,wo​)=∫Ω+​Li​(x,wi​)fr​(x,wi​,wo​)cosθdwi​=∫A​Li​(x,wi​)fr​(x,wi​,wo​)∣∣x′−x∣∣2cosθcosθ′​dA

# Sampling the Light

把着色点的光照计算分成两个部分

  • 光源对着色点的影响（直接光照，不需要计算 RR 概率）。
  • 其他物体反射后对着色点的影响（间接光照，需要计算 RR）。

# 渲染方程

shade(p,wo)shade(p, wo)shade(p,wo)：

  • 在光源上均匀采样，并计算光线亮度 Llight=Li∗fr∗cos⁡θ∗cos⁡θ′/∣∣x′−p∣∣2/pdf(light)L_{light} = L_i * f_r * \cos \theta * \cos \theta^{\prime} / || x^{\prime} - p||^2 / pdf(light)Llight​=Li​∗fr​∗cosθ∗cosθ′/∣∣x′−p∣∣2/pdf(light) 。

  • 最后对两者求和 Llight+LotherL_{light} + L_{other}Llight​+Lother​。

# 最终的渲染方程：

对于被遮挡的光线，直接不计算即可

shade(p,wo)shade(p, wo)shade(p,wo)：

  • 在光源上均匀采样，并求和。
    • 如果光线不被遮挡：计算光线亮度 Llight=Li∗fr∗cos⁡θ∗cos⁡θ′/∣∣x′−p∣∣2/pdf(light)L_{light} = L_i * f_r * \cos \theta * \cos \theta^{\prime} / || x^{\prime} - p||^2 / pdf(light)Llight​=Li​∗fr​∗cosθ∗cosθ′/∣∣x′−p∣∣2/pdf(light) 。
