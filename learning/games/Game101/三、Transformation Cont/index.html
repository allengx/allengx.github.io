# 为什么 RyR_yRy​ 看起来是反的？

因为 x，y，z 之前的循环对称关系，导致，其中总会有一组坐标的相乘是反向的。

x^×y^=z^\hat x \times \hat y = \hat zx^×y^​=z^

y^×z^=x^\hat y \times \hat z = \hat xy^​×z^=x^

z^×x^=−y^\hat z \times \hat x = - \hat yz^×x^=−y^​

x^×z^=y^\hat x \times \hat z = \hat yx^×z^=y^​

# 观测变换（Viewing transformation）

  • Model（模型）transformation：把场景的物体摆放好
  • View（视图）/ Camera transformation：把摄像机放在一个合适的位置
  • Projection（投影）
    • Orthographic（正交）
    • Perspective（透视）

# 视图📷

  • Position： e⃗\vec ee
  • Look-at / gaze direction：g⃗\vec gg​
  • Up direction：t⃗\vec tt

# 观察的关键

  • 相机和物体的距离
  • 相机的角度

只要保证上述两点，不论何地，最终的画面就不会改变。所以通常情况下，会把相机放在「原点坐标」。

# 正交（Orthographic）投影

  • 相机 lookat： −z⃗-\vec z−z，up：y⃗\vec yy​

  • Z 坐标就可以丢弃了～🙌

    • 如何区分前后呢❓

# 透视（Perspective）投影

  • 如何表示透视投影下的坐标点：任意三维坐标都可以转为(xz,yz,z2,z!=0)(xz , yz , z^2 , z!=0)(xz,yz,z2,z!=0)
    • 远处的物体，投影到近处以后，大小会随着 Z 的变化而变化 (1,0,0,1)=(2,0,0,2)(1,0,0,1) = (2,0,0,2)(1,0,0,1)=(2,0,0,2)
